# 📋 SELA 交易 API 修復記錄

> **日期**: 2026-01-17  
> **版本**: v0.8.3  
> **狀態**: ✅ 已修復

---

## 問題概述

新增台股/美股交易時，連續遇到三個 HTTP 錯誤：

```
405 Method Not Allowed → 422 Unprocessable Entity → 500 Internal Server Error
```

---

## 問題 1：405 Method Not Allowed

### 現象
```
POST /api/portfolio/transactions/tw HTTP/1.1" 405 Method Not Allowed
```

### 原因分析

| 項目 | 內容 |
|------|------|
| **後端路由定義** | `@router.post("/transactions")` |
| **前端請求路徑** | `/api/portfolio/transactions/tw` ❌ |
| **問題** | 前端多加了 `/tw` 和 `/us` 後綴 |

### 錯誤程式碼 (transaction.js)
```javascript
// 台股
res = await apiRequest('/api/portfolio/transactions/tw', { method: 'POST', body });
// 美股
res = await apiRequest('/api/portfolio/transactions/us', { method: 'POST', body });
```

### 修復方式
```javascript
// 統一使用不帶後綴的路徑
res = await apiRequest('/api/portfolio/transactions', { method: 'POST', body });
```

---

## 問題 2：422 Unprocessable Entity

### 現象
```
POST /api/portfolio/transactions HTTP/1.1" 422 Unprocessable Entity
```

### 原因分析

| 項目 | 內容 |
|------|------|
| **後端 Schema** | `market: str = Field(..., pattern="^(tw\|us)$")` (必填) |
| **前端 body** | 沒有 `market` 欄位 ❌ |

### 錯誤程式碼 (transaction.js)
```javascript
const body = {
    symbol,
    name,
    transaction_type: type,  // 缺少 market!
    quantity,
    price,
    // ...
};
```

### 修復方式
```javascript
// 台股
const body = {
    symbol,
    name,
    market: 'tw',  // ✅ 新增
    transaction_type: type,
    quantity,
    price,
    // ...
};

// 美股
const body = {
    symbol,
    name,
    market: 'us',  // ✅ 新增
    transaction_type: type,
    quantity,
    price,
    // ...
};
```

## 問題 3：500 Internal Server Error

### 現象
```
POST /api/portfolio/transactions HTTP/1.1" 500 Internal Server Error
TypeError: PortfolioService.create_transaction() got an unexpected keyword argument 'broker_id'
```

### 原因分析

| 項目 | 內容 |
|------|------|
| **Router 層** | 傳入 `broker_id=data.broker_id` |
| **Service 層** | `create_transaction()` 方法沒有 `broker_id` 參數 ❌ |

### 錯誤程式碼 (portfolio_service.py)
```python
async def create_transaction(
    self,
    user_id: int,
    symbol: str,
    market: str,
    # ... 其他參數
    note: Optional[str] = None,
    # 缺少 broker_id!
) -> PortfolioTransaction:
```

### 修復方式
```python
async def create_transaction(
    self,
    user_id: int,
    symbol: str,
    market: str,
    # ... 其他參數
    note: Optional[str] = None,
    broker_id: Optional[int] = None,  # ✅ 新增
) -> PortfolioTransaction:
    
    # 建立交易紀錄時也要傳入
    transaction = PortfolioTransaction(
        # ... 其他欄位
        note=note,
        broker_id=broker_id,  # ✅ 新增
    )
```

## 修復檔案清單

| 檔案 | 修復內容 |
|------|---------|
| `static/js/transaction.js` | 移除 API 路徑的 `/tw`、`/us` 後綴 |
| `static/js/transaction.js` | 台股 body 加入 `market: 'tw'` |
| `static/js/transaction.js` | 美股 body 加入 `market: 'us'` |
| `app/services/portfolio_service.py` | `create_transaction()` 加入 `broker_id` 參數 |

## 經驗總結

### HTTP 錯誤對照表

| 錯誤碼 | 含義 | 常見原因 |
|--------|------|----------|
| 405 | Method Not Allowed | 路徑不匹配、HTTP 方法錯誤 |
| 422 | Unprocessable Entity | Schema 驗證失敗、缺少必填欄位 |
| 500 | Internal Server Error | 後端程式錯誤、參數不匹配 |

### 新功能開發檢查清單

當新增需要前後端配合的欄位時，確認以下都已同步：

- [ ] 後端 Pydantic Schema (TransactionCreate)
- [ ] 後端 SQLAlchemy Model (PortfolioTransaction)
- [ ] 後端 Service 方法參數
- [ ] 前端 API 請求 body
- [ ] 資料庫遷移 (如有新增欄位)

### 除錯技巧

# 查看 Railway 即時日誌

cat logs.json | python3 -c "import json,sys; [print(l['message']) for l in json.load(sys.stdin) if '422' in l['message'] or '405' in l['message'] or '500' in l['message']]"
